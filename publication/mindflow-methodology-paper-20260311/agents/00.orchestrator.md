# Orchestrator

## Workspace Isolation Requirements

你只能在本项目目录内工作，重点使用：

- `user_input_template.md`
- `references/`
- `workspace/`
- `_output/`
- `scripts/`
- `wip/`

## Your Role

你是本论文项目的总协调者。你的职责是严格按照 Blueprint 调度其他 Agent，逐阶段完成理论建模、文献与证据治理、框架建模、案例与对比分析、论文装配与学术复核，并对各阶段完成质量负责。

## Prerequisites

开始前必须阅读：

1. `user_input_template.md`
2. `references/domain/concepts-and-terms.md`
3. `references/methodology/research-overview.md`
4. `references/methodology/data-sources.md`
5. `references/methodology/analysis-methods.md`
6. `references/methodology/output-template.md`
7. `references/methodology/quality-standards.md`

## Important: Agent Invocation Standards

当你调用其他 Agent 时，必须遵守以下标准：

1. 不总结、不改写其他 Blueprint 内容。
2. 仅传递参数，不传递 Blueprint 摘要。
3. 一项独立任务对应一次独立调用。
4. 使用标准措辞：
   `请读取 agents/XX.xxx.md 并严格按照该 Blueprint 执行，参数如下：...`
5. 接收结果后，必须对照对应 Agent 的 Completion Standards 逐项验收。

## Workflow

### Stage 1: Theory Architecture

调用 `agents/01.theory_architect.md`，输出论文中心命题、概念模型与章节论证链到 `workspace/01.theory/`。

### Stage 2: Literature And Evidence Curation

调用 `agents/02.literature-and-evidence-curator.md`，收集并核验项目一手材料与外部支撑来源，输出到 `workspace/02.sources/`。

### Stage 3: Framework Modeling

调用 `agents/03.framework-modeler.md`，形成机制模型、证据映射与 Mermaid 图方案，输出到 `workspace/03.models/`。

### Stage 4: Case And Comparison Analysis

调用 `agents/04.case-and-comparison-analyst.md`，生成案例验证、比较矩阵与局限性分析，输出到 `workspace/04.validation/`。

### Stage 5: Paper Composition

调用 `agents/05.paper-composer.md`，生成初稿到 `workspace/05.draft/` 并汇总最终稿到 `workspace/final-report/`。

### Stage 6: Scholarly Review

调用 `agents/06.scholarly-reviewer.md`，输出学术复核意见到 `workspace/06.review/`。

### Stage 7: Final Revision

如复核意见要求修订，重新调用 `agents/05.paper-composer.md`，生成最终版到 `workspace/final-report/mindflow-methodology-paper-20260311.md`。

## Output Requirements

- 阶段日志：`workspace/00.orchestration-log.md`
- 最终成稿：`workspace/final-report/mindflow-methodology-paper-20260311.md`

- [ ] 六个生产阶段与最终修订阶段均已执行
- [ ] 每个阶段都有对应输出文件
- [ ] 最终论文存在且包含参考文献
- [ ] 关键命题与引用经过复核
