# Analysis Methods Guide

## Analysis Framework

本项目采用“理论建模 + 框架形式化 + 案例验证 + 范式对比”的综合方法。

## Core Questions

围绕下列分析主轴展开：

1. 问题断裂：现有范式缺什么
2. 结构定义：`Mind` 七层各自承担什么职责
3. 约束机制：`Soul` 和心智模型如何进入运行时
4. 推衍机制：`Inference` 如何支撑未知条件下的判断与学习方向
5. 知识机制：知识如何被内化为 `Capacity`
6. 软件范式：知识可执行如何改写软件的结构边界
7. 对比结论：MindFlow 的方法论增量是什么

## Analysis Process

### Stage 1: 提取项目一手材料

从 MindFlow 仓库中提取与理论命题直接相关的设计说明、协议文本和目录结构信息。

### Stage 2: 收集外部支撑来源

围绕知识工程、知识管理、MAS、认知架构、AI 软件等方向收集可引用资料。

### Stage 3: 建立命题与证据映射

把每一条论文核心命题绑定到具体证据与论证材料。

### Stage 4: 构建章节级论证链

形成从问题提出到方法论、再到验证和讨论的闭合结构。

### Stage 5: 输出图表与正文

按论文结构写作，并插入服务论证的 Mermaid 图与对比表。

## Quality Requirements

- 每一条强结论都必须有来源或明确的论证基础
- 对预测性结论必须显式标注边界
- 引用、概念与图表之间保持一致
- 技术描述不能脱离理论目的
