# task-learning

This directory stores task-level `Learning Record`s. It is the task-level candidate knowledge area.

## File Naming Rule

`tl-{task-id}.md`

## Template and Rules

See `TEMPLATE.md` for the formal structure, required fields, and validation rules.

## Fixed Rules

- one `Task` may have only one primary `Learning Record`
- not readable by `Learning(Read)` (see `mind/learning/learning-read/README.md §Read Scope`)
- content must first enter `drafts/`, then enter `reviews/`
- `tl-{task-id}.md` must always be written as part of terminal `Learning`, even when `reflection-report.md` has no learning candidates; in that case, write `Candidate Knowledge: none` and `Next Promotion Target: none`
