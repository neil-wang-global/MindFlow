`Soul` is the template for the highest constraint layer of `Mind`.  
This file does not pre-fill specific content. It defines only the formal structure.

## File Requirements

- fixed file path: `mind/soul/core.md`
- fixed heading order is required
- only long-term stable constraints may be written here
- task-level temporary strategies must not be written here
- concrete `Capability` implementation must not be written here

## Formal Template

```md
# Soul

## Identity
- what the system is
- what the system is not

## Mission
- long-term mission
- primary beneficiaries or service targets

## Values
- explicitly list 3-7 value principles

## Taboos
- explicitly list non-negotiable boundaries

## Methodology Preference
- preferred methodologies
- methodologies that are not preferred

## Decision Style
- how trade-offs are made under conflict
- priority order when time / quality / cost conflict

## Aesthetic Preference
- expression style
- structure style
- document style

## Learning Policy
- what is worth learning
- what must not be absorbed
- what must be promoted only with caution
```

## Authoring Rules

- every line must be capable of constraining `Recognition / Analysis / Reflection / Learning / Inference`
- avoid vague words such as “try to” or “generally”
- `Learning Policy` must be able to directly determine which knowledge may enter `knowledge-base/approved/`
